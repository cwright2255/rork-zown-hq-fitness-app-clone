# EAS Update Preview Workflow

This repo has a GitHub Actions workflow at `.github/workflows/eas-preview.yml` that publishes a JavaScript-only preview update to Expo's update service whenever you push to `feat/fitleus-light-theme-codemod` or open a pull request against `main`. The workflow then comments a QR code and a deep link directly on the PR so you can preview the change in **Expo Go** in seconds 芒聙聰 no rebuild required.

## What the workflow does

1. Checks out the branch and installs Node 20 + Bun.
2. Installs the Expo + EAS CLIs via [`expo/expo-github-action@v8`](https://github.com/expo/expo-github-action), authenticated with the `EXPO_TOKEN` repo secret.
3. From inside `expo/`:
   - `bun install --frozen-lockfile`
   - `eas update --branch <auto-derived name> --message <auto-derived message> --non-interactive`
4. On pull-request events, runs `expo/expo-github-action/preview@v8` so the EAS update appears as a sticky comment with a QR code and an `exp://` deep link.

The branch name on Expo is derived automatically:
- **Push events:** the Git ref name with `/` replaced by `-` (so `feat/fitleus-light-theme-codemod` becomes the EAS branch `feat-fitleus-light-theme-codemod`).
- **Pull requests:** `pr-<number>`.

EAS auto-creates the channel and branch on the first `eas update` call 芒聙聰 no manual setup is required on the Expo side.

## Previewing the update on a phone

Once the workflow runs successfully:

1. Open the PR on GitHub.
2. Look for the comment from `expo-github-action[bot]` titled "Preview". It contains a QR code and an `exp://` URL.
3. On iOS or Android, install the **Expo Go** app from the App Store / Play Store (free).
4. Open the camera app (iOS) or the Expo Go app (Android) and scan the QR code in the PR comment.
5. The new JS bundle is downloaded and the app loads with the codemod applied. Total time from `git push` to a preview on your phone is usually 2芒聙聯4 minutes.

You can also tap the deep link from your phone if you're reading the PR there.

## Limits 芒聙聰 when this isn't enough

EAS Update only ships **JavaScript and assets**. It does **not** rebuild native code. The following changes still require a development build (`eas build --profile development`):

- New native modules added via `expo install` or `yarn add`.
- Changes to `app.json` keys that affect the native runtime (`ios.bundleIdentifier`, `android.package`, splash screen images, native permissions, etc.).
- Upgrading the Expo SDK major version.

For pure UI/JS changes 芒聙聰 including this codemod 芒聙聰 `eas update` is the fast path.
